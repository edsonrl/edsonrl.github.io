Projects

These are select projects that I've worked on in the last years.

  • [current] Storage Data Flow Management based on Machine Learning (SMML) 1.

    The objective of this project is to automate data flow management for caching and storage tiering in hybrid data storage solutions, while relying on newly-developed artificial intelligent algorithms. The proposed methodology involves combining mathematical modeling with streaming machine learning for the first time, for guiding the decisions of data storage systems.

  • Self-Tuning for SQL-on-Hadoop Systems (PhD) 1, 2, 3, 4.

    The objective was to make SQL-on-Hadoop systems self-tuning. I extended Starfish (a tuning advisor for Hadoop) to support Hive queries. Integrated Starfish into the Hive’s query processor to automatically generate and apply tuning setups at fine-grained level. This new approach materializes 5.3x less data, reduces the number of straggler tasks by 5x, decreases CPU usage in 35% and memory usage in 40%, and reduces job profiling time in 50%.

  • Testing Orders of Magnitude (TOOM) 1, 2.

    The objective of the TOOM project was to stress test relational databases with shifting workloads. We developed a stress testing tool for DBMSs called Model-based Database Stress Testing (MoDaST). The new method uncover bugs in portions of the code that are only reached with non-functional testing. At the time, MoDaST found a major race condition in VoltDB (ENG-6881).
